## 一、初步调研结果

- WebSocket是一种基于TCP的全双工通信协议，通过单个长连接实现客户端与服务器之间的双向实时数据传输。

#### 典型应用场景：实时聊天、协同编辑、实时数据看板、多人在线互动等。

## 二、详细调研报告

# WebSocket协议调研报告

## 1. 协议概述

WebSocket是一种在单个TCP连接上进行全双工通信的网络协议，由IETF标准化为RFC 6455。它位于应用层，基于TCP协议，旨在解决HTTP协议在实时、双向通信场景下的不足。

传统HTTP模式下，客户端必须不断轮询服务器或使用长轮询来获取新数据，这会带来高延迟、大量无效请求和头部开销。WebSocket的出现使得服务器可以主动向客户端推送数据，从而大幅提升实时交互效率。

## 2. 与HTTP协议的对比

| 维度 | HTTP | WebSocket |
|------|------|-----------|
| **连接模式** | 短连接（或长轮询伪长连） | 真正长连接，一次握手后持久保持 |
| **通信方式** | 半双工，客户端请求→服务器响应 | 全双工，客户端与服务器可同时发送数据 |
| **传输开销** | 每次请求携带完整HTTP头部，数据帧开销大 | 数据帧头部仅2～14字节，轻量高效 |
| **适用场景** | 静态资源获取、RESTful API | 实时聊天、实时推送、协同编辑、游戏同步 |

> **核心区别**：WebSocket通过HTTP协议完成连接升级后，便脱离了HTTP的请求/响应模式，转为基于帧的全双工通信，避免了反复握手和冗余头部。

## 3. 工作原理

- **握手建立过程**：客户端发送一个带有`Upgrade: websocket`和`Connection: Upgrade`头的HTTP请求，服务器返回`101 Switching Protocols`状态码，协议即从HTTP升级为WebSocket，之后在同一TCP连接上传输WebSocket数据帧。

- **数据帧传输机制**：通信以帧为单位，支持文本和二进制数据。帧格式包含操作码（如文本、二进制、关闭、Ping/Pong等）、掩码标志、有效载荷长度等，保证数据边界清晰。

- **长连接保持与断开**：通过Ping/Pong帧保持心跳，防止中间代理或防火墙断开空闲连接。任意一方可发送关闭帧（opcode 8）来优雅关闭连接，TCP连接随后释放。

## 4. 核心特性

- **全双工通信**：客户端和服务器可独立、同时发送消息，无需等待对方响应。
- **低延迟**：连接建立后，无需反复握手，服务器可即时推送，延迟远低于轮询。
- **数据轻量**：帧头部极小，单次通信开销显著低于HTTP的重复头部。
- **兼容跨域**：WebSocket原生支持跨域，不受同源策略限制，通过服务器回传的`Sec-WebSocket-Accept`键保证安全性。
- **事件驱动模型**：在浏览器和小程序中，API均采用事件回调机制，便于异步处理。

## 5. 典型应用场景

- **实时聊天**：如微信小程序中的在线客服、直播间弹幕。
- **实时数据看板**：股票行情、传感器监控、运维大屏等。
- **在线互动**：多人在线游戏、白板协作、投票抢答。
- **协同编辑**：多人同时编辑文档，操作实时广播。
- **推送通知**：服务器主动向小程序推送订单状态、系统消息等。

## 6. 微信小程序中的应用

微信小程序对WebSocket提供了完整的API支持，关键接口如下：

- `wx.connectSocket({ url })` – 发起WebSocket连接，需指定`wss://`协议（加密传输）。
- `wx.onSocketOpen` – 监听连接成功事件，之后可进行通信。
- `wx.sendSocketMessage({ data })` – 发送文本或二进制数据。
- `wx.onSocketMessage` – 接收服务器推送的消息。
- `wx.closeSocket()` / `wx.onSocketClose` – 主动关闭连接及监听关闭事件。
- `wx.onSocketError` – 处理异常断开等错误。

**典型调用流程**：

```javascript
const socketTask = wx.connectSocket({
  url: 'wss://example.com/chat'
});
socketTask.onOpen(() => {
  socketTask.send({ data: 'hello' });
});
socketTask.onMessage((res) => {
  console.log('收到消息：', res.data);
});
```

课堂中我们强调，`wx.connectSocket`返回的`SocketTask`对象可更精确地管理该连接（如监听、发送、关闭），适合复杂场景。这与原生WebSocket API设计理念一致，体现了移动端对实时交互的支持能力。

## 7. 调研总结

**优势**：WebSocket实现了真正意义上的服务器推送，极大降低了实时应用的延迟和带宽消耗，API简洁统一，跨平台兼容性好。

**局限性**：需要服务器单独维持长连接，对服务器并发能力有要求；部分网络环境（如严格防火墙）可能阻断WebSocket流量；在小程序中必须使用`wss://`，增加了证书部署成本。

**移动端实际价值**：在微信小程序生态中，WebSocket是实现实时聊天、协同、数据驱动界面更新的关键手段。通过合理使用心跳、重连机制和状态管理，可以构建出流畅的实时互动体验，是移动开发工程师必须掌握的核心能力之一。
