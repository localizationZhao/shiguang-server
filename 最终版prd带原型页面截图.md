# 食光 · 产品需求文档（PRD）

> 像素便签风美食社交微信小程序 | 个人全栈开发

---

## 文档说明

本项目采用 **底部 4 Tab 一级架构 + 二级子页面 + 弹窗浮层** 结构，围绕 **DIY 菜谱创作、公共菜谱浏览、虚拟餐厅点餐、个人数据管理** 四大模块。

---

## 1. 整体架构

### 1.1 一级导航

| Tab | 名称 | 定位 |
|-----|------|------|
| 🔧 DIY | 自制菜谱 | 用户私有菜谱的存储、分类管理、创作主界面 |
| 🏠 首页 | 查菜谱 | 公共菜谱库，浏览参考 + 一键复刻 |
| 🍽 餐厅 | 餐厅 | 虚拟餐厅 + 好友点餐（差异化社交核心） |
| 👤 我的 | 个人中心 | 数据聚合 + 快捷入口 + 系统设置 |

### 1.2 核心业务链路

```
公共菜谱浏览 → 一键复刻 → DIY 改编/自创 → 上架餐厅 → 好友扫码加入 → 点单 → 接单/出餐 → 订单归档
```

### 1.3 页面层级

- 一级 Tab × 4
- 二级子页面：菜谱详情、菜谱编辑、订单详情、做菜记录、收藏列表、我的菜谱/订单/餐厅、管理员配图
- 弹窗/浮层：创建餐厅、加入餐厅、分类管理、导出分享、管理员验证

---

## 2. Tab1：DIY 自制菜谱

### 2.1 主页面

**左侧分类栏**：
- 7 个系统分类：荤菜、素菜、凉菜、汤羹、主食、甜点、酒水
- 自定义分类：新增、改名、删除（菜谱归入"未分类"）
- 点击分类 → 右侧实时筛选

**右侧卡片区**：
- 像素便签风卡片（4:3 比例，双层 box-shadow 边框）
- 5 色可选：粉/蓝/橘/紫/绿
- 展示 emoji 封面、菜名、价格、分类标签

**搜索**：支持菜名、食材名搜索

**批量管理**：
- 多选菜谱
- 聚合食材 → 生成采购清单 → 一键复制到剪贴板

**长按菜单**：编辑 / 上架餐厅 / 删除（二次确认）

### 2.2 新建/编辑菜谱（5 步表单）

| 步骤 | 内容 |
|------|------|
| 第 1 步 | 名称 + 分类 + 定价 |
| 第 2 步 | 食材清单（7 类：肉类/水产/蔬菜/豆制品/水果/调味料/其他辅料），可增删行 |
| 第 3 步 | 烹饪步骤，每步限 500 字，支持步骤配图 + 拖拽排序 |
| 第 4 步 | 参考资料链接（选填） |
| 第 5 步 | 封面（emoji 或拍照/相册，1:1 裁剪）+ 5 色选择 → 保存 |

支持草稿暂存，部分填写即可保存。草稿不显示在 DIY 列表中。

### 2.3 导出

菜谱详情页 → 导出菜单：
- 选择聊天分享
- 复制链接
- 纯文字列表导出
- 以长图导出
- 以 SQL 语句导出（可发给好友直接导入数据库）

## 3. Tab2：首页 / 查菜谱

### 3.1 主页面

- **搜索**：菜名、食材名关键词
- **标签筛选**：早餐/午餐/晚餐/肉类/蔬菜/甜品/海鲜/主食/汤类/小吃/蛋奶类（11 标签）
- **双列瀑布流**：emoji + 菜名 + 价格 + 分类
- 数据来源：本地 PUBLIC_RECIPES 秒显 + 云端静默更新

### 3.2 盲盒转盘

解决"不知道吃什么"：
- 点击触发 1.5s 旋转动画
- `randomPick()` 随机抽取一道菜
- 弹窗展示结果，可"再抽一次"或查看详情

### 3.3 菜谱详情页

- 封面（emoji 或图片）+ 基本信息卡
- 食材清单 + 分步烹饪步骤（支持步骤图预览）
- 收藏 / 一键复刻（深拷贝至 DIY，source: 'copy'） / 分享

### 3.4 灵宠自定义

- 3 套系统预设宠物形象
- 拍照或相册选图自定义
- 5 套主题色

## 4. Tab3：餐厅

### 4.1 权限规则

| 角色 | 限制 |
|------|------|
| 店主 | 每人最多创建 5 个餐厅 |
| 食客 | 每人最多加入 3 个餐厅 |
| 身份 | 可同时拥有店主 + 食客双重身份，一键切换 |

### 4.2 创建与加入

**创建餐厅**：名称（限 8 字）→ 自动生成邀请码 `SG + 时间戳(36进制)` + 24h 有效期 → 云端同步。

**加入餐厅**：
- 输入邀请码 / 扫小程序码
- 选鸟种（18 种）+ 选空座（8 座，4 号为店主专属主座）+ 选配饰（13 种）+ 灵魂光晕（20 色）
- 离线模式：仅支持本地已有餐厅

### 4.3 餐厅内部空间

8 座虚拟餐位：上座、雅座、散座、主座（店主专属）、边座。成员以像素鸟形象落座。

### 4.4 店主端

- 菜单管理：上架/下架 DIY 菜谱
- 订单管理：待接单 → 接单（制作中）→ 完成出餐 / 拒单
- 消息：快捷回复预设（"别着急""五分钟就好啦"等）+ 自定义消息（限 25 字）
- 成员管理：查看在线状态，可移出成员

### 4.5 食客端

- 浏览菜单 → 勾选下单
- 实时追踪订单状态
- 催单预设（"快写吧您内""我看您才是米其林大厨"等）
- 订单完成后可评分（1-5 星 + 文字评价）

### 4.6 订单状态流转

```
pending（待接单）→ cooking（制作中）→ done（已出餐）→ 已评价
                                    ↘ rejected（已拒单）
```

### 4.7 感受分享（Feeds）

- 文字 + 多图
- 可见性：所有人 / 餐厅内 / 仅自己
- 位置：精确 / 模糊（前 6 字 + "附近"）/ 自定义
- 时间：公开 / 隐藏

### 4.8 闭店

店主删除 → 软删除标记 `closed` → 所有加入副本（按 `originalId` 和 `inviteCode` 双重匹配）同步标记 → 食客触发闭店通知 → 可"不再怀念"清理数据。

### 4.9 分享

邀请码 + 小程序码（粉色 `#ffa3cb` 线条，`wxacode.getUnlimited` 生成）。

## 5. Tab4：我的

### 5.1 页面结构

- 个人信息：头像、昵称、ID、生日
- 数据统计：自制菜谱数、做菜次数、订单总数、收藏数

便签风瀑布流卡片 + 5 色可选 + 照片上传（4:3 裁剪）+ 语音录音（60s，MP3）+ 文字笔记。云端同步。

### 5.3 小鸟设置

5 种显示模式：全页面 / 自定义（逐页开关）/ 仅餐厅 / 仅内部空间 / 关闭。

### 5.4 数据修复

一键重置本地全部数据，恢复系统分类。用于数据异常时自助修复。

## 6. 口袋小鸟

独立 Canvas 2D 组件，跨页面持久化。

- **18 物种**：蓝鸟、红雀、金翅雀、燕子、鸽子、长尾山雀、欧亚鸲、蓝松鸦、卡罗莱纳鹪鹩、家朱雀等
- **行为状态机**：跳跃→飞行→栖息→被抚摸（自动切换）
- **交互**：拖拽定位 / 轻点抚摸 + 随机气泡文字 / 长按弹出 **birbOS 菜单**（切换物种、换帽子、冻结位置、隐藏小鸟、查看收集进度）
- **收集系统**：羽毛掉落（15% 稀有 + 85% 常见）→ 解锁新物种；帽子随机掉落 → 换装
- **音效**：`wx.createWebAudioContext()` 正弦波振荡器模拟鸟鸣
- **持久化**：位置、物种、帽子、冻结态、隐藏态跨页面保存

## 7. 像素风 UI 规范

**核心技法**：CSS box-shadow 双层边框：

```css
box-shadow:
  3rpx 0 var(--hl), -3rpx 0 var(--hl),   /* 内层左右 */
  6rpx 0 var(--hl), -6rpx 0 var(--hl),   /* 外层左右 */
  0 0 0 3rpx var(--hl),                  /* 缝隙填充 */
  0 0 0 6rpx #fff;                       /* 白色间隙 — 像素感关键 */
```

配色：奶油底 `#ffecda` + 5 套主题色（绿/蓝/粉/橘/紫），双重 box-shadow + border-radius 形成像素凸起感。

## 8. 技术实现要点

| 特性 | 方式 |
|------|------|
| 离线可用 | 全量 `wx.Storage` 本地缓存，云端异步同步 |
| 类型安全 | `data-id` 全站 `Number()` 转换 |
| 超时控制 | 云调用 8s 超时 + 离线 fallback（管理员接口 30s） |
| 邀请码 | 24h 有效期 `SG + Date.now().toString(36)` |
| 扫码进店 | `wxacode.getUnlimited` 生成带 scene 参数码 |
| 数据去重 | `relatedIds` Set 匹配餐厅关联副本 |
| 餐厅切换 | `_lastRestId` ID 匹配，避免索引错位闪跳 |
| 复刻 | 深拷贝 → 新 ID → `addRecipe()` 本地 → `api.addRecipe({is_public:0})` 云端 |
| 图片处理 | 1:1 裁剪 + 压缩 + `wx.cloud.uploadFile` 云存储 |
| 闭店 | `closed` 标记 + `inviteCode`/`originalId` 双重匹配关联副本 |
