**产品整体架构与分页面拆解（PRD）**

文档说明

本产品采用底部 4Tab 一级架构 + 二级子页面 + 弹窗浮层的三级页面体系，围绕 DIY 菜谱创作、公共菜谱参考、虚拟餐厅点餐、个人数据管理四大核心模块，严格适配原型交互逻辑与整体产品设计规范。

1\. 产品整体架构

1.1 一级导航骨架

小程序底部固定常驻 4 个一级 Tab，从左至右依次为：

Tab1 DIY：用户私有菜谱的创作与管理中心

Tab2 首页 / 查菜谱：系统公共菜谱库，提供学习参考与创作素材

Tab3 餐厅：虚拟餐厅与好友点餐，社交差异化核心模块

Tab4 我的：个人数据聚合中心与设置入口

1.2 核心业务链路

参考公共菜谱 → DIY 改编 / 自创菜谱 → 菜谱上架至虚拟餐厅 → 好友在线点餐 → 店主接单 / 制作完成 → 订单归档留存 → 做菜记录沉淀。

产品定位：美食创作 + 居家烹饪 + 熟人社交的全链路闭环工具（非商业用途）。

1.3 页面层级结构

一级 Tab（4 个）

二级子页面（菜谱详情 / 编辑、餐厅详情、做菜记录等）

弹窗 / 浮层（创建餐厅、分类筛选、消息通知等）

系统级组件（全局搜索、分享卡片、空状态引导等）

统一交互规范：全产品统一页面跳转动画、点击反馈、状态刷新、弹窗提示的交互逻辑，确保操作体验一致，降低用户学习成本。

2\. 分页面详细架构拆解

2.1 Tab1：DIY 自制菜谱（产品核心创作模块）

2.1.1 页面定位

用户私有菜谱的存储、分类管理、新增创作的主页面，承载产品核心 DIY 创作能力。

2.1.2 页面分区与功能说明

（1）顶部操作区

页面居中配置【新建菜谱】快捷按钮，点击进入新建菜谱流程。

按钮样式突出，符合视觉规范中的主色调与圆角规范。

（2）左侧分类栏

系统预置分类：荤菜、素菜、凉菜、汤羹、主食、甜点、酒水。

用户自定义能力：新增分类（自定义名称）、修改分类名称、删除分类（删除后该分类下菜谱归入 "未分类"）、长按拖拽排序。

交互反馈：点击分类标签，右侧菜谱卡片区实时筛选展示对应菜谱。

（3）中部菜谱卡片区

卡片信息：菜品封面图、菜名、售价（可为 0）、所属分类标签。

单击：进入菜谱详情 / 编辑页。

长按 / 更多按钮：唤起功能菜单栏，包含编辑菜谱、删除菜谱（二次确认弹窗）、上架至餐厅。

批量操作模式（右上角 "管理" 按钮进入）：

支持多选菜谱，支持批量导入食材采购清单（聚合所选菜谱的全部食材，去重展示）。

（4）空状态页面

触发条件：用户无任何自制菜谱。

展示内容：浅灰色引导文案 "快来做你的第一道菜吧"，下方配新建菜谱引导按钮。

2.1.3 二级页面：新建 / 编辑菜谱

采用分步表单设计，降低单次填写压力，支持草稿暂存，操作更灵活。

第 1 步 基本信息：菜谱名称、所属分类标签、菜品定价，支持新建 / 删除分类，全部可暂存草稿。

第 2 步 配方信息：食材清单（肉类、水产、蔬菜、豆制品、水果、调味料、其他辅料），分项填写，支持增删行。

第 3 步 烹饪步骤：分步录入烹饪步骤，单步骤限 500 字，支持配图，支持拖拽排序步骤，支持上传步骤实拍图。

第 4 步 参考资料：参考链接（非必填），文本输入框。

第 5 步 封面与保存：上传菜品封面图（拍照 / 相册，需请求系统权限），确认创建。允许仅填写部分步骤即可保存；保存后自动存入 DIY 菜谱库，支持后续二次编辑。

2.1.4 菜谱导出与分享

导出格式：PDF、图片、Word。

分享方式：生成分享链接，好友可在线查看菜谱内容。

权限说明：导出 / 分享的菜谱为快照版本，对方不可修改源内容。

2.2 Tab2：首页 / 查菜谱（系统公共菜谱库）

2.2.1 页面定位

平台内置公共菜谱的展示与检索模块，为用户提供学习参考与二次创作素材。用户仅可查看、收藏、一键复刻，无权修改官方源菜谱内容。

2.2.2 页面分区与功能说明

（1）顶部搜索区

全局搜索框，支持以下维度精准检索：菜名关键词、食材名称、菜系标签、口味关键词。

搜索结果以列表形式展示，支持与分类筛选组合使用。

（2）盲盒推荐区

核心痛点：解决用户 "不知道吃什么" 的选择困难。

交互形态：悬浮小转盘或简易灵宠形象，点击触发随机抽取。

灵宠自定义：用户可上传图片 / 拍照自定义灵宠脸部形象，可从系统预设的几套颜色方案中选择外观。

推荐逻辑：用户选择偏好标签（早餐、午餐、晚餐、蔬菜、肉类、甜品等），系统在对应数据范围内随机抽取菜谱推荐。

（3）菜谱筛选与展示区

筛选方式：弹窗式分类筛选面板。

左侧：食材大类。

右侧：菜系标签（点击食材大类后关联展示）。

支持组合筛选，精准匹配。

展示方式：瀑布流布局，展示精选优质菜谱卡片。

2.2.3 菜谱详情页

完整展示内容：食材清单、用料配比、分步烹饪做法、成品效果图。

一键复刻：点击后系统自动复制菜谱内容至用户个人 DIY 菜谱列表（菜谱模板与 DIY 一致），用户可自由二次自定义修改、创新。

收藏功能：收藏数据实时同步至个人中心 "我的收藏" 列表。

分享功能：支持在线分享给好友。

2.3 Tab3：餐厅（虚拟餐厅与好友点餐）

2.3.1 页面定位

产品核心差异化社交模块，用户可自主创建专属虚拟餐厅，支持加入好友餐厅，实现熟人线上点菜趣味互动。

2.3.2 权限规则

自建餐厅：单人最多创建 1 家。

加入餐厅：单人最多加入 3 个好友餐厅。

双重身份：用户可同时拥有 "店主" 和 "食客" 两种身份。

2.3.3 页面状态说明

（1）无餐厅空白页

触发条件：用户未创建餐厅且未加入任何好友餐厅。

展示内容：居中引导文案："快来创建属于自己的餐厅吧～"，核心操作按钮【创建餐厅】，底部提示文案：支持通过好友分享链接加入他人餐厅。

创建餐厅弹窗：填写餐厅名称、上传餐厅头像，提交即可开店。

（2）已有餐厅常态页

顶部切换区：支持多餐厅标签切换，包含个人自建餐厅、已加入的好友餐厅。

订单管理及消息区（店主端与食客端双视角）：

店主：查看待接单 / 制作中 / 已完结全部订单；执行接单、拒单、标记出餐完成；设置预估上菜时间；发送快捷消息或自定义消息给下单用户。

食客：浏览餐厅上架菜单、勾选菜品下单；实时查看订单进度；接收店主消息、发送催促消息。

菜品管理区：店主可上架 / 下架自己 DIY 菜谱中的菜品，已上架菜品可被食客浏览和下单。

感受分享区（可选显示）：

定位：唯一公共社交区域。

隐私控制：用户可在 "我的→设置" 中关闭此功能显示。

发布内容：文字、图片、表情、不超过 2 分钟的短视频，可选择是否附带餐厅名称和昵称。

发布形式：做菜记录或进食记录分享。

可见范围：发布时可选择 "所有人可见" 或 "仅餐厅成员可见"。

互动：平台用户可互相评论。

2.3.4 餐厅分享

系统自动生成专属餐厅分享链接与二维码。

场景一（加入模式）：好友点击分享链接，占用加入名额正式入店。

场景二（访客模式）：访客扫码可临时单点点餐，无需加入餐厅。

2.4 Tab4：我的（个人中心）

2.4.1 页面定位

全产品用户数据的聚合总入口，汇总个人全部创作、餐厅、订单、社交数据，集成各类功能快捷入口与系统设置。

2.4.2 页面分区与功能说明

（1）顶部信息区

展示用户头像、昵称、账号 ID。

（2）中部数据统计区

核心数据可视化展示，数据实时同步更新：自制菜谱总数、做菜总次数、历史订单总数。

（3）功能快捷入口

做菜记录：独立二级页面，可记录下厨时间、绑定对应菜谱、上传成品实拍图。

我的餐厅：快速跳转至个人自建餐厅。

订单记录：汇总全部历史订单，支持按状态筛选查看。

我的收藏：展示收藏的公共菜谱列表。

在线客服：客服咨询入口。

（4）设置列表

个人资料修改：修改头像、昵称等信息。

隐私设置：控制感受分享区可见性等隐私选项。

帮助中心：使用指南与常见问题。

关于产品：版本信息与产品介绍。

异常数据修复：数据异常时的自助修复工具。

3\. 非功能需求

产品全面满足以下四大核心要求：

易用性：统一交互规范，分步表单降低操作门槛，空状态均有明确引导。

运行稳定性：本地缓存策略减少云端请求频次，图片自动压缩优化加载速度。

设备兼容性：基于 uni-app 框架，适配微信小程序全场景。

数据安全性：用户私人菜谱权限隔离，精细化角色权限控制，未上架菜品完全私密。
